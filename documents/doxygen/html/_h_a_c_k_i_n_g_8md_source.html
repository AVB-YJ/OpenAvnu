1 # Contributing
2 
3 If you have a bug fixed or added support for some AVTP feature, your patches
4 are welcomed! In order to get your patches merged faster, please follow the
5 guidelines:
6 
7 * Check if all patches are following the coding style from libavtp project. See
8 section 'Coding Style' for more information.
9 
10 * Before submitting your patch to review, make sure it doesn't break any unit
11 test. See section 'Running Unit Tests' for information about how to build and
12 run unit tests.
13 
14 * Besides the bugfix/feature itself, also provide unit test covering the code
15 you're contributing. See section 'Code Coverage' to check how you can easily
16 generate coverage reports and see where you need to work on to get your code
17 covered.
18 
19 * If your patch adds new public APIs to libavtp, please also provide patches
20 adding example applications (or modify an existing one if it makes sense)
21 which demonstrate how to use the new APIs.
22 
23 * Make sure the author's name and email are set properly.
24 
25 # Coding Style
26 
27 The coding style from libavtp is pretty much the same from Linux kernel
28 described in https://www.kernel.org/doc/html/latest/process/coding-style.html.
29 
30 The style for line wrapping is to indent as far as possible to the right
31 without hitting the 80 columns limit.
32 
33 Example:
34 
35 ```
36 /* Correct */
37 int avtp_aaf_pdu_set(struct avtp_stream_pdu *pdu, enum avtp_aaf_field field,
38  uint64_t val)
39 
40 /* Wrong */
41 int avtp_aaf_pdu_set(struct avtp_stream_pdu *pdu, enum avtp_aaf_field field,
42  uint64_t val)
43 ```
44 
45 # Running Unit Tests
46 
47 Unit tests required libcmocka so make sure you have cmocka packages installed
48 in your system.
49 
50 Run the following command to build and run all unit tests. Make sure the build
51 system files are generated already, see section 'Build' in the README.md file
52 for more information.
53 
54 ```
55 $ ninja -C build test
56 ```
57 
58 # Code Coverage
59 
60 Meson build system provides some built-in code coverage support based on `lcov`
61 so make sure you have this package installed on your system in order to
62 generate coverage reports.
63 
64 To generate html reports run the following commands:
65 
66 ```
67 $ rm -rf build/
68 $ meson build -Db_coverage=true
69 $ ninja -C build/ test
70 $ ninja -C build/ coverage-html
71 ```
72 
73 The coverage report can be found in build/meson-logs/coveragereport/ directory.

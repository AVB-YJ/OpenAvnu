1 avdecc-lib
2 ==========
3 
4 Simple C++ library for implementing IEEE1722.1 (AVB Device Enumeration, Discovery and Control)
5 
6 
7 Introduction
8 ------------
9 
10 This library aims to simplify development of an AVDECC controller based on the IEEE1722.1 specification.
11 It provides a simple c++ object interface to 1722.1 objects and implements device discovery and enumeration
12 as a background process.
13 
14 The repository contains source to build a Windows DLL or a Linux library and a command line application for
15 exercising the library.
16 
17 The overall philosophy of AVDECC LIB is to implement a thin layer of commands that allow an application to
18 discover and and control AVDECC capable endpoints. The internal operations of the library are designed to be single threaded,
19 although multiple threads are used to queue operations to be performed by the single threaded "engine" portion of the library.
20 The library supports notification events (callbacks) that are triggered on the success (or failure) of a command.
21 It is up to the application to process the notifications in a useful manner. Asynchronously control updates from an
22 endpoint are also supported. A control notification does not have data about the updated descriptor values embedded
23 in it. Instead the AVDECC application should query the control class to obtain the updated values.

97 Operations
98 ==========
99 
100 AVDECC Endpoint Discovery
101 -------------------------
102 
103 When the AVDECC system receives a AVDECC advertise message from an endpoint, it proceeds to
104 enumerate the endpoint's complete object model if it hasn't done so already and the following hold true:
105 
106 * The AEM_SUPPORTED flag in the ADP entity_capabilities field is set
107 * The GENERAL_CONTROLLER_IGNORE flag in the ADP entity_capabilities field is not set
108 * The ENTITY_NOT_READY flag in the ADP entity_capabilities field is not set
109 
110 When an end station is discovered, a notification message is sent to the application. Additionally,
111 upon completion of the enumeration process, a second notification message is sent to the application.
112 
113 *AL: Is there a case for separating the discovery and enumeration notifications? The user may want to connect entities that do not support AEM*
114 *AGE: changed - see above*

116 AVDECC descriptor reads
117 -----------------------
118 
119 A descriptor read by referencing the object the object of interest. Since the AVDECC system has
120 already read all descriptors, the read operation is completed without producing any network traffic.
121

123

148 AVDECC commands
149 ---------------
150 
151 An AVDECC command is sent to the target object, ie
152 
153  istream = avdeccsys->endpoint(0)->entity(0)->configuration(0)->stream_input(0);
154  if (istream) {
155  istream->set_format(unsigned int *id,format,...);
156  }
157 
158 Completion results in a notification message of success or failure via the callback mechanism.

220 Callbacks

222

226  void avdecc_notify(void *user_obj, int32_t notification, int32_t endpoint_index, int32_t obj, int32_t index, int32_t cmd, int32_t id);

228 With notifications of:
229 * No Match Found
230 * Endpoint Discovered
231 * Endpoint Connected
232 * Endpoint Disconnected
233 * Invalid Command
234 * Command Timeout
235 * Command Resent
236 * Command Success

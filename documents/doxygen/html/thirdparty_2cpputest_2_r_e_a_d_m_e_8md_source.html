17 ## Getting Started
18 
19 You'll need to do the following to get started:
20 
21 Building from source (unix-based, cygwin, MacOSX):
22 
23 * Download latest version
24 * autogen.sh
25 * configure
26 * make
27 * make check
28 * You can use "make install" if you want to install CppUTest system-wide
29 
30 You can also use CMake, which also works for Windows Visual Studio.
31 
32 * Download latest version
33 * cmake CMakeList.txt
34 * make
35 
36 Then to get started, you'll need to do the following:
37 * Add the include path to the Makefile. Something like:
38  * CPPFLAGS += -I(CPPUTEST_HOME)/include
39 * Add the memory leak macros to your Makefile (needed for additional debug info!). Something like:
40  * CXXFLAGS += -include $(CPPUTEST_HOME)/include/CppUTest/MemoryLeakDetectorNewMacros.h
41  * CFLAGS += -include $(CPPUTEST_HOME)/include/CppUTest/MemoryLeakDetectorMallocMacros.h
42 * Add the library linking to your Makefile. Something like:
43  * LD_LIBRARIES = -L$(CPPUTEST_HOME)/lib -lCppUTest -lCppUTestExt
44 
45 After this, you can write your first test:
46 
47 ```C++
48 TEST_GROUP(FirstTestGroup)
49 {
50 };
51 
52 TEST(FirstTestGroup, FirstTest)
53 {
54  FAIL("Fail me!");
55 }
56 ```

58 ## Command line switches
59 
60 * -v verbose, print each test name as it runs
61 * -r# repeat the tests some number of times, default is one, default if # is not specified is 2. This is handy if you are experiencing memory leaks related to statics and caches.
62 * -g group only run test whose group contains the substring group
63 * -n name only run test whose name contains the substring name
64 
65 ## Test Macros
66 
67 * TEST(group, name) - define a test
68 * IGNORE_TEST(group, name) - turn off the execution of a test
69 * TEST_GROUP(group) - Declare a test group to which certain tests belong. This will also create the link needed from another library.
70 * TEST_GROUP_BASE(group, base) - Same as TEST_GROUP, just use a different base class than Utest
71 * TEST_SETUP() - Declare a void setup method in a TEST_GROUP - this is the same as declaring void setup()
72 * TEST_TEARDOWN() - Declare a void setup method in a TEST_GROUP
73 * IMPORT_TEST_GROUP(group) - Export the name of a test group so it can be linked in from a library. Needs to be done in main.
74 
75 ## Set up and tear down support
76 
77 * Each TEST_GROUP may contain a setup and/or a teardown method.
78 * setup() is called prior to each TEST body and teardown() is called after the test body.
79 
80 ## Assertion Macros
81 
82 The failure of one of these macros causes the current test to immediately exit
83 
84 * CHECK(boolean condition) - checks any boolean result
85 * CHECK_TRUE(boolean condition) - checks for true
86 * CHECK_FALSE(boolean condition) - checks for false
87 * CHECK_EQUAL(expected, actual) - checks for equality between entities using ==. So if you have a class that supports operator==() you can use this macro to compare two instances.
88 * STRCMP_EQUAL(expected, actual) - check const char* strings for equality using strcmp
89 * LONGS_EQUAL(expected, actual) - Compares two numbers
90 * BYTES_EQUAL(expected, actual) - Compares two numbers, eight bits wide
91 * POINTERS_EQUAL(expected, actual) - Compares two const void *
92 * DOUBLES_EQUAL(expected, actual, tolerance) - Compares two doubles within some tolerance
93 * FAIL(text) - always fails
94 * TEST_EXIT - Exit the test without failure - useful for contract testing (implementing an assert fake)
95 
96 
97 Customize CHECK_EQUAL to work with your types that support operator==()
98 
99 * Create the function: `SimpleString StringFrom(const yourType&)`
100 
101 The Extensions directory has a few of these.

103 ## Building default checks with TestPlugin
104 
105 * CppUTest can support extra checking functionality by inserting TestPlugins
106 * TestPlugin is derived from the TestPlugin class and can be inserted in the TestRegistry via the installPlugin method.
107 * TestPlugins can be used for, for example, system stability and resource handling like files, memory or network connection clean-up.
108 * In CppUTest, the memory leak detection is done via a default enabled TestPlugin
109 
110 Example of a main with a TestPlugin:
111 
112 ```C++
113 int main(int ac, char** av)
114 {
115  LogPlugin logPlugin;
116  TestRegistry::getCurrentRegistry()->installPlugin(&logPlugin);
117  int result = CommandLineTestRunner::RunAllTests(ac, av);
118  TestRegistry::getCurrentRegistry()->resetPlugins();
119  return result;
120 }
121 ```
122 
123 Memory leak detection
124 
125 * A platform specific memory leak detection mechanism is provided.
126 * If a test fails and has allocated memory prior to the fail and that memory is not cleaned up by TearDown, a memory leak is reported.
127  It is best to only chase memory leaks when other errors have been eliminated.
128 * Some code uses lazy initialization and appears to leak when it really does not (for example: gcc stringstream used to in an earlier release). One cause is that some standard library calls allocate something and do not free it until after main (or never).
129  To find out if a memory leak is due to lazy initialization set the -r switch to run tests twice. The signature of this situation is that the first run shows leaks and the second run shows no leaks. When both runs show leaks, you have a leak to find.
130 
131 ## How is memory leak detection implemented?
132 
133 * Before setup() a memory usage checkpoint is recorded
134 * After teardown() another checkpoint is taken and compared to the original checkpoint
135 * In Visual Studio the MS debug heap capabilities are used
136 * For GCC a simple new/delete count is used in overridden operators new, new[], delete and delete[]
137 
138 If you use some leaky code that you can't or won't fix you can tell a TEST to ignore a certain number of leaks as in this example:
139 
140 ```C++
141 TEST(MemoryLeakWarningTest, Ignore1)
142 {
143  EXPECT_N_LEAKS(1);
144  char* arrayToLeak1 = new char[100];
145 }
146 ```

159 ## Example Test
160 
161 ```C++
162 #include "CppUTest/TestHarness.h"
163 #include "ClassName.h"
164 
165 TEST_GROUP(ClassName)
166 {
167  ClassName* className;
168 
169  void setup()
170  {
171  className = new ClassName();
172  }
173  void teardown()
174  {
175  delete className;
176  }
177 }
178 
179 TEST(ClassName, Create)
180 {
181  CHECK(0 != className);
182  CHECK(true);
183  CHECK_EQUAL(1,1);
184  LONGS_EQUAL(1,1);
185  DOUBLES_EQUAL(1.000, 1.001, .01);
186  STRCMP_EQUAL("hello", "hello");
187  FAIL("The prior tests pass, but this one doesn't");
188 }
189 ```

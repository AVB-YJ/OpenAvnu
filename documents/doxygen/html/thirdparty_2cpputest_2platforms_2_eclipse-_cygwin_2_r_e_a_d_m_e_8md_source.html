1 ## Using the Eclipse-Cygwin Project
2 
3 This project is for use on the Windows platform. These are the steps required for using it. (If you only wish to compile the CppUTest libraries, it is recommended that you use the method described at http://cpputest.github.io, rather than setting up Eclipse).
4 
5 ### Preparation
6 #### 1. Install Cygwin
7 You need to have Cygwin installed, with at least the Gnu C\C++ compiler, make, autotools and libtool. Please follow the instructions at http://cpputest.github.io to build CppUTest from the Cygwin bash prompt.
8 
9 #### 2. Set your PATH
10 Next, you need to add the path to your Cygwin binaries to your Windows system path, e.g.
11 ```dos
12 C:<path_to_Cygwin>\Cygwin\bin
13 ```
14 
15 #### 3. Install Eclipse CDT
16 Use your existing Eclipse CDT (Juno, Kepler, ...) or unpack the release to your system drive. You may use the 32 bit version as it will work on all systems. Unless you have a specific reason, you do not require the 64 bit version. You may need to install or update your Java JRE and add it to your system path. The JRE needs to match, e.g. 32 bit Eclipse requires the 32 bit JRE.
17 
18 #### 4. Install the C/C++Unit plugin
19 Open Eclipse.
20 Accept the default workspace for now (you may want to set a more appropriate workspace location later on). Then install the "C/C++ Unit Test" plugin:
21  "Help"->"Install New Software"-> work with "Kepler - http://download.eclipse.org/releases/kepler".
22  You will find the plugin under:
23 ```
24  [ ] Programming Languages
25  [x] C/C++ Unit Testing Support.
26 ```
27 While you are at it, you might also want to install:
28 ```
29  [ ] Linux Tools
30  [x] Gcov Integration
31 ```
32 
33 #### 5. Add the CppUTest plugin
34 Clone the [CppUTest Eclipse Test Runner](https://github.com/tcmak/CppUTestEclipseJunoTestRunner) and add it to Eclipse following the instructions there.
35 
36 ### Import this project into Eclipse
37 In Eclipse you can't simply "open" a project - you have to first import it into the workspace:
38  * File->Import...->General->Existing Projects into workspace
39  * [Next >]->Select root directory: `<cpputest_dir>/Platforms/Eclipse-Cygwin`
40  * Make sure that `[ ] Copy projects into workspace` is **NOT** ticked.
41  * Click Finish.
42 
43 ### Compile and run tests
44 Before you can compile CppUTest in Eclipse, you must configure it using Automake. Instructions can be found at http://cpputest.github.io/. Once you have done this, select a configuration (Libraries, Check, CppUTestTests or CppUTestExtTests) via Project->BuildConfiguration->Set Active>. The 'Check' configuration will build the libraries along with all tests and run the tests.
45 
46 ### Working with the C/C++ Unit plugin
47 Make sure your unit test executable has been built and exists, e.g. cpputest_build/CppUTestTests.exe.
48 
49 The first time you run tests using the plugin, you need to select them via
50 Run->Run Configurations...->C/C++Unit->CppUTestTest.
51 Click [Run]. (If Run is greyed out, you need to build the configuration CppUTestTests first)
52 Once you have run the tests at least once, you should be able to select them by clicking the down triangle next to the green "Run" icon, where it should be listed right at the top. On subsequent runs, the executable will be built automatically, if necessary.
53 
54 Note that launch configurations have been included with this project; in your own projects, you will need to create these yourself.
55 
56 This is an example of what a successful test run would look like:
57 ![Successful Test Run](https://raw.githubusercontent.com/cpputest/cpputest.github.io/master/images/eclipse-testrunner-01.png)
58 You can select and rerun individual tests in the upper pane, and error messages for failed tests will appear in the lower pane.
59 ### Troubleshooting
60 #### Problem: I am getting no test results...
61 ...but you know your tests should have run, and you did not receive any error message.
62 ##### Possible Reason:
63 This can happen if the Cygwin-generated binary can't find the Cygwin DLLs. (If you were to run your test executable at the Windows cmd prompt, you would see a Windows error message box to that effect). Since no error message is displayed in the console via stderr, this leaves the test runner ignorant of what happenend
64 ##### Solution:
65 You need to add <path_to_Cygwin>/Cygwin/bin to your system PATH variable. Then you must restart Eclipse.

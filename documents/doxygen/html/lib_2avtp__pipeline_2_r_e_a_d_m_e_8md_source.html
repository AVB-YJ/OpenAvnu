2 
3 ## General Status
4 - Consider the AVTP Pipeline a work in progress.
5 - Integrated with OpenAvnu:
6  - gPTP
7  - MAAP
8  - MSRP
9  - credit based shaper
10  - build system
11 - Credit-based shaper algorithm is not used for individual streams.
12 - Testing of various mappings and benchmarking has been performed. Look at the end of this file for benchmark results.
13 - Documentation and doc generation has not been fully updated.

15 ## Building Current OpenAvnu
16 ### Tool chain and libraries
17 - Ubuntu 14.04
18 - Install dependencies for OpenAvnu ($ sudo apt-get install ...)
19  - $ sudo apt-get install build-essential
20  - $ sudo apt-get install libpcap-dev
21  - $ sudo apt-get install libpci-dev
22  - $ sudo apt-get install libsndfile1-dev
23  - $ sudo apt-get install libjack-dev
24  - $ sudo apt-get install linux-headers-generic
25  - linux-headers (same version as the kernel you're building for)
26 
27 - Additional install dependencies for AVTP pipeline
28  - $ sudo apt-get install libglib2.0-dev
29  - $ sudo apt-get install libasound2-dev
30 
31 - Additional install dependencies for AVTP pipeline if GStreamer support is enabled
32  - $ sudo apt-get install libgstreamer0.10-dev
33  - $ sudo apt-get install libgstreamer-plugins-base0.10-dev

63 ## Running OpenAvnu daemons
64 - Helper scripts in the repo root.
65 - `$ sudo ./run_igb.sh eth1`
66  - Load the igb driver. Supply the interface name (ethx) as parameter. Only needed if using IGB support for the Intel i210 adapter.
67 - `$ sudo ./run_daemons.sh eth1`
68  - Start the gptp, msrp, maap, and shaper daemons. Supply the interface name (ethx) as parameter.
69  - Daemons can also be started individually using the run_gptp.sh, run_srp.sh, run_maap.sh, and run_shaper.sh scripts.
70 - `$ sudo ./stop_daemons.sh`
71  - Stop the gptp, msrp, maap, and shaper daemons. Don't use this command while AVTP Pipeline is running.
72 
73 ## Running OpenAvnu AVTP Pipeline example
74 - `$ sudo ./run_avtp_pipeline.sh eth1`
75  - Run the current OpenAvnu AVTP Pipeline example. Supply the interface name (ethx) as parameter.
76 - `$ sudo ./stop_avtp_pipeline.sh eth1`
77  - Stop the current OpenAvnu AVTP Pipeline example. The script will also attempt to cleanly recover if the AVTP pipeline binaries crashed.
78 
79 The AVTP Pipeline example is expected to be run simultaneously on two or more different Linux computers,
80 with the network interfaces connected using AVB-capable switches.
81 (The daemons do not currently support using two different network interfaces on the same computer, so different computers must be used.)
82 You can refer to the list of [Avnu Certified Products](http://avnu.org/certified-products/) for switches with AVB/TSN support.
83 
84 To connect the Talker and Listener with the example implementation, you need to use an AVDECC controller.
85 (These are also referred to as 1722.1 or ATDECC controllers. AVDECC was renamed to ATDECC by the IEEE P1722.1 work group in 2017.)
86 This will tell the Listener(s) which stream to listen to,
87 and allow the Talker and Listener(s) to coordinate when they should start streaming.
88 There are several AVDECC controllers available, including one in the OpenAvnu avdecc-lib/controller folder.
89 
90 The AVTP Pipeline example Talker and Listener should also be compatible with other AVB/TSN products that support 8-channel,
91 48K/24-bit [IEC 61883-6](https://webstore.iec.ch/preview/info_iec61883-6%7Bed2.0%7Den.pdf) audio and AVDECC management.
92 The list of [Avnu Certified Products](http://avnu.org/certified-products/) includes some of them.
93 The example Talker and Listener has also been used successfully to stream audio to and from Apple Macbooks running macOS version 10.12 (Sierra) and later,
94 and controlled with the Apple Macbook built-in AVDECC controller (avbutil).

137  REPORT=0 # statistics display interval in seconds (0 turns off statistics)
